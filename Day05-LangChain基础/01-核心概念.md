## 二、@tool 装饰器

```python
from langchain.tools import tool

@tool
def multiply(a: int, b: int) -> int:
    """将两个整数相乘。

    Args:
        a: 第一个整数
        b: 第二个整数
    """
    return a * b

@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气"""
    return f"{city}：晴，25°C"

# 工具的 docstring 就是 LLM 看到的描述！
# 必须写清楚，LLM 靠它决定调用哪个工具
```

## 三、create_agent —— 创建 Agent

> 注意：`create_agent` 是 langchain 0.3+ 的新 API。如果你的版本没有这个函数，
> 可以用 `langgraph.prebuilt.create_react_agent` 替代（功能相同，更推荐）。

```python
from langchain.agents import create_agent
from langchain.tools import tool

@tool
def add(a: int, b: int) -> int:
    """两数相加"""
    return a + b

@tool
def multiply(a: int, b: int) -> int:
    """两数相乘"""
    return a * b

agent = create_agent(
    model="openai:gpt-4o",    # provider:model 格式
    tools=[add, multiply],
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "3+4等于几？再乘以2"}]
})
print(result["messages"][-1].content)
```

### 结构化输出

```python
from pydantic import BaseModel, Field
from langchain.agents import create_agent
from langchain.agents.structured_output import ProviderStrategy

class Weather(BaseModel):
    temperature: float
    condition: str

agent = create_agent(
    model="openai:gpt-4o",
    tools=[get_weather],
    response_format=ProviderStrategy(Weather),
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "北京天气？"}]
})
print(result["structured_response"])  # Weather(temperature=25.0, condition='晴')
```

### 长期记忆

```python
from dataclasses import dataclass
from langchain.agents import create_agent
from langchain.tools import ToolRuntime, tool
from langgraph.store.memory import InMemoryStore

@dataclass
class Context:
    user_id: str

store = InMemoryStore()
store.put(("users",), "user_123", {"name": "小明", "lang": "中文"})

@tool
def get_user_info(runtime: ToolRuntime[Context]) -> str:
    """查询用户信息"""
    user_id = runtime.context.user_id
    user = runtime.store.get(("users",), user_id)
    return str(user.value) if user else "未知用户"

agent = create_agent(
    model="openai:gpt-4o",
    tools=[get_user_info],
    store=store,
    context_schema=Context,
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "查一下用户信息"}]},
    context=Context(user_id="user_123"),
)
```

## 四、LCEL（LangChain Expression Language）

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_template("讲一个关于{topic}的笑话")
model = ChatOpenAI()
chain = prompt | model | StrOutputParser()

result = chain.invoke({"topic": "猫"})
```

### 管道操作符 `|`

```python
# 等价于
output_parser.invoke(
    model.invoke(
        prompt.invoke({"topic": "猫"})
    )
)
```

### 常用组件

```python
# Prompt 模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个{role}"),
    ("human", "{question}"),
])

# 输出解析器
StrOutputParser()            # 字符串
JsonOutputParser()           # JSON
PydanticOutputParser(Model)  # Pydantic 模型

# 模型
ChatOpenAI(model="gpt-4o", temperature=0)

# 组合
chain = prompt | model | StrOutputParser()
result = chain.invoke({"role": "助手", "question": "你好"})
```

### 批处理

```python
# 批量调用
results = chain.batch([
    {"topic": "猫"},
    {"topic": "狗"},
    {"topic": "鱼"},
])
```

### 流式

```python
# 流式输出
for chunk in chain.stream({"topic": "猫"}):
```

## 五、RAG 基础

### 什么是 Embedding？

Embedding 是将文本映射为高维向量（如 1536 个浮点数），向量之间的距离代表语义相似度。

```
"退款" → [0.12, -0.34, 0.56, ...]  ┐ 距离近（语义相似）
"退钱" → [0.11, -0.35, 0.55, ...]  ┘

"退款" → [0.12, -0.34, 0.56, ...]  ┐ 距离远（语义不同）
"天气" → [-0.45, 0.78, -0.12, ...] ┘
```

为什么需要？关键词搜索无法理解语义——搜"退款"找不到包含"退钱"的文档，但 Embedding 可以。

```python
from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_community.vectorstores import FAISS
from langchain.chains import RetrievalQA

# 1. 加载文档
loader = PyPDFLoader("document.pdf")
documents = loader.load()

# 2. 分块（chunk_size 太大→检索不精准，太小→丢失上下文，一般 500-1500）
splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200,  # 重叠部分保持语义连续性
)
chunks = splitter.split_documents(documents)

# 3. 嵌入 + 向量存储
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = FAISS.from_documents(chunks, embeddings)
# FAISS 是 Facebook 开源的向量检索库，纯内存，适合开发
# 生产环境可换 Milvus、Qdrant、Pinecone 等

# 4. 检索链
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})  # 取最相似的 3 个块
llm = ChatOpenAI(model="gpt-4o")
chain = RetrievalQA.from_chain_type(llm=llm, retriever=retriever)

# 5. 查询
result = chain.invoke("文档的主要内容是什么？")
```

## 六、面试高频问题

**Q: LangChain 和 LangGraph 的关系？**
A: LangChain 提供基础组件（@tool、LCEL、模型集成）。LangGraph 提供图编排能力（替代旧 AgentExecutor）。生产环境推荐 LangGraph。

**Q: LCEL 是什么？为什么引入？**
A: 用 `|` 管道符组合 Runnables。提供声明式的流式、异步、批处理、重试/回退能力。

**Q: RAG 的基本流程？**
A: 加载文档 → 分块 → 嵌入 → 存入向量库 → 检索相关块 → 传给 LLM 生成答案。

**Q: 工具的 docstring 为什么重要？**
A: LLM 靠工具的描述（docstring）决定调用哪个工具。描述不清楚会导致 LLM 选错工具。
