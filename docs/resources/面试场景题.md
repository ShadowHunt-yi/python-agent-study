# Agent 面试场景题

> 开放性系统设计题，面试中常见。每个题目给出回答框架。

## 一、设计一个客服 Agent

**题目**：设计一个客服 Agent，支持多轮对话、工具调用、人工介入。

**回答框架**：

```
1. 架构选择
   - LangGraph StateGraph 编排
   - 状态：messages、customer_info、ticket_status、escalated

2. 工具设计
   - query_order: 查询订单
   - refund: 退款操作
   - create_ticket: 创建工单
   - escalate_to_human: 转人工

3. 关键流程
   - 意图识别 → 路由到对应处理节点
   - 退款等敏感操作需要 interrupt + 人工确认
   - 多轮对话通过 Checkpointing 保持上下文

4. 安全考虑
   - 敏感操作需要人工审批
   - 日志记录所有操作
   - 退款金额限制

5. 扩展
   - 接入知识库（RAG）回答常见问题
   - 多语言支持
   - 满意度调查
```

## 二、设计一个代码 Review Agent

**题目**：设计一个能自动 Review PR 的 Agent。

**回答框架**：

```
1. 触发方式
   - GitHub Webhook → FastAPI 接收
   - PR 创建/更新时触发

2. 工具设计
   - read_diff: 获取 PR diff
   - read_file: 读取完整文件
   - run_linter: 运行 linter
   - run_tests: 运行测试
   - add_comment: 添加 review 评论

3. Review 流程
   - 读取 diff → 逐文件分析
   - 检查：代码风格、安全漏洞、性能问题、逻辑错误
   - 生成结构化 review 意见

4. 输出格式
   - 总体评价
   - 具体问题（带行号）
   - 改进建议

5. 技术栈
   - OpenAI SDK 调用 LLM
   - LangGraph 编排多步 Review
   - FastAPI 接收 Webhook
   - Pydantic 校验输入输出
```

## 三、设计一个多 Agent 协作系统

**题目**：设计一个系统，多个 Agent 协作完成研究报告。

**回答框架**：

```
1. Agent 角色
   - Supervisor: 任务分配和协调
   - Researcher: 搜索和整理信息
   - Writer: 撰写报告
   - Reviewer: 审阅和修改

2. 协作模式
   - Supervisor 模式：主管分配任务
   - 流水线：Researcher → Writer → Reviewer
   - 反馈循环：Reviewer 不满意 → Writer 修改

3. 共享状态
   - 研究资料（共享 Store）
   - 报告草稿（State）
   - Review 意见（消息传递）

4. 技术实现
   - LangGraph StateGraph
   - 共享 InMemoryStore
   - interrupt 人工审阅节点

5. 质量控制
   - 事实核查（Researcher 交叉验证）
   - 一致性检查（Reviewer）
   - 引用来源标注
```

## 四、设计一个 Agent 的记忆系统

**题目**：如何为 Agent 设计长期记忆？

**回答框架**：

```
1. 记忆类型
   - 短期记忆：对话历史（messages 列表）
   - 长期记忆：用户偏好、历史交互
   - 工作记忆：当前任务中间状态

2. 存储方案
   - 短期：内存 / Redis
   - 长期：向量数据库（语义检索）+ KV 存储（精确查询）
   - 工作：LangGraph State

3. 记忆管理
   - 写入：对话结束后提取关键信息
   - 检索：语义搜索 + 时间衰减
   - 遗忘：定期清理过期记忆

4. 技术实现
   - Embedding 存入向量库
   - LangGraph Store
   - 摘要压缩长对话

5. 隐私考虑
   - 数据加密
   - 用户可删除记忆
   - 合规性（GDPR）
```

### 5.1 手写 ReAct 循环

```python
# 题目：实现一个简单的 ReAct 循环（不用框架）
async def react_loop(query: str, tools: dict, max_steps: int = 5) -> str:
    messages = [{"role": "user", "content": query}]
    for step in range(max_steps):
        response = await call_llm(messages, tools=list(tools.keys()))
        if not response.tool_calls:
            return response.content
        for tc in response.tool_calls:
            result = tools[tc.name](tc.args)
            messages.append({"role": "tool", "content": result, "tool_call_id": tc.id})
    return "达到最大步数"
```

### 5.2 手写带重试的 LLM 调用

```python
# 题目：实现带指数退避重试的 LLM 调用
async def call_with_retry(client, messages, max_retries=3, timeout=30):
    for attempt in range(max_retries):
        try:
            return await asyncio.wait_for(
                client.chat.completions.create(model="gpt-4o", messages=messages),
                timeout=timeout,
            )
        except (RateLimitError, asyncio.TimeoutError):
            if attempt == max_retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)
```

## 六、概念题

**Q: Agent 和 Chatbot 的区别？**
A: Chatbot 只能对话。Agent 能调用工具、做决策、执行多步任务。

**Q: 怎么防止 Agent 陷入死循环？**
A: 设置 max_steps 限制、检测重复工具调用、超时机制。

**Q: 怎么评估 Agent 的质量？**
A: 轨迹评估（步骤是否合理）、结果评估（答案是否正确）、在线指标（用户满意度、任务完成率）。

**Q: MCP 是什么？**
A: Model Context Protocol，Anthropic 提出的标准化协议，让 Agent 通过统一接口连接外部工具和数据源。
