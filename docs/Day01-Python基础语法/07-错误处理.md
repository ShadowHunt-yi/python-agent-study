# 07 — 错误处理

## 一、try/except/finally/else

```python
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"除零错误: {e}")
except (ValueError, TypeError) as e:
    print(f"值/类型错误: {e}")
except Exception as e:
    print(f"其他错误: {e}")
else:
    print("没有异常时执行")
finally:
    print("总是执行")
```

```javascript
// JS 对比
try {
    const result = 10 / 0;
} catch (e) {
    if (e instanceof RangeError) {
        console.log(`Error: ${e.message}`);
    } else {
        throw e;
    }
} finally {
    console.log("总是执行");
}
```

### 关键差异

| 差异 | Python | JS |
|------|--------|-----|
| 捕获关键字 | `except` | `catch` |
| 抛出关键字 | `raise` | `throw` |
| 多类型捕获 | `except (TypeA, TypeB)` | 需要多个 catch 或 if |
| else 子句 | 有（无异常时执行） | 无 |
| 类型捕获 | `except TypeError` 直接写 | `if (e instanceof TypeError)` |

### except 顺序

```python
# 先捕获具体异常，再捕获通用异常
try:
    risky_operation()
except FileNotFoundError:
    print("文件不存在")
except PermissionError:
    print("权限不足")
except OSError:
    print("其他 OS 错误")
except Exception:
    print("兜底")
```

## 二、raise 抛出异常

```python
# 抛出异常
raise ValueError("无效输入")
raise TypeError("期望 int，得到 str")

# 重新抛出当前异常
try:
    risky_operation()
except ValueError:
    print("记录日志")
    raise                # 重新抛出原始异常

# 异常链（Python 3+）
try:
    open("missing.txt")
except FileNotFoundError as original:
    raise RuntimeError("初始化失败") from original
```

```javascript
// JS
throw new Error("Invalid input");
throw new TypeError("Expected number");
```

## 三、自定义异常

```python
# 继承 Exception（不是 BaseException）
class ValidationError(Exception):
    """校验失败"""
    pass

class NotFoundError(Exception):
    def __init__(self, resource: str, resource_id: int):
        self.resource = resource
        self.resource_id = resource_id
        super().__init__(f"{resource} with id {resource_id} not found")

# 使用
raise ValidationError("名称不能为空")
raise NotFoundError("User", 42)
```

```javascript
// JS
class ValidationError extends Error {
    constructor(message) {
        super(message);
        this.name = "ValidationError";
    }
}
```

## 四、常见内置异常

| 异常 | 触发场景 | 示例 |
|------|---------|------|
| `ValueError` | 类型对，值不对 | `int("abc")` |
| `TypeError` | 类型完全不对 | `"hi" + 42` |
| `KeyError` | dict key 不存在 | `{}["missing"]` |
| `IndexError` | list 索引越界 | `[][0]` |
| `AttributeError` | 属性/方法不存在 | `None.length` |
| `FileNotFoundError` | 文件不存在 | `open("nope.txt")` |
| `ImportError` | 导入失败 | `import nonexistent` |
| `StopIteration` | 迭代器耗尽 | `next(iter([]))` |
| `RuntimeError` | 通用运行时错误 | `asyncio.run()` 嵌套 |
| `PermissionError` | 权限不足 | 写只读文件 |

## 五、上下文管理器 with

> `with` 和 `as` 是 Python 管理资源的语法，离开代码块时自动清理。完整讲解见 09-文件与模块.md 和 Day02-上下文管理器。

```python
# 最常见的用法 —— 文件操作
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()
# 离开 with 块时自动关闭文件

# 多个上下文管理器
with open("in.txt") as fin, open("out.txt", "w") as fout:
    fout.write(fin.read())
```

### 自定义上下文管理器

```python
# 方式一：类
class Timer:
    def __enter__(self):
        import time
        self.start = time.time()
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        import time
        self.elapsed = time.time() - self.start
        print(f"耗时: {self.elapsed:.3f}s")
        return False  # 不抑制异常

with Timer() as t:
    # 一些耗时操作
    pass
print(f"实际耗时: {t.elapsed}s")

# 方式二：contextmanager 装饰器
from contextlib import contextmanager

@contextmanager  # @ 是装饰器语法，详见 Day02-装饰器
def timer():
    import time
    start = time.time()
    try:
        yield start       # yield 暂停函数并返回值，详见 Day02-生成器与迭代器
    finally:
        elapsed = time.time() - start
        print(f"耗时: {elapsed:.3f}s")

with timer():
    pass
```

## 六、异常处理最佳实践

```python
# ✓ 捕获具体异常
try:
    value = int(user_input)
except ValueError:
    print("请输入数字")

# ✗ 不要裸 except
try:
    risky()
except:               # 捕获一切，包括 KeyboardInterrupt
    pass

# ✓ 用 finally 清理资源
resource = acquire()
try:
    use(resource)
finally:
    release(resource)

# ✓ 用 EAFP 原则（Easier to Ask Forgiveness than Permission）
# Python 风格：先做，出错再处理
try:
    value = my_dict[key]
except KeyError:
    value = default

# ✗ LBYL（Look Before You Leap）—— 不够 Pythonic
if key in my_dict:
    value = my_dict[key]
else:
    value = default

# ✓ 但 get() 更好
value = my_dict.get(key, default)
```

## 七、assert 断言（仅用于调试）

```python
# assert 在优化模式下会被跳过，不要用于业务逻辑
assert x > 0, f"x 必须大于 0，当前值: {x}"

# 等价于
if not x > 0:
    raise AssertionError(f"x 必须大于 0，当前值: {x}")
```

## 八、面试高频问题

**Q: try/except 的 else 和 finally 有什么区别？**
A: else 在没有异常时执行；finally 无论是否有异常都执行（用于清理资源）。

**Q: 为什么要捕获具体异常？**
A: 避免隐藏 bug。裸 `except` 会吞掉所有异常，包括你没预料到的错误。

**Q: EAFP 和 LBYL 是什么？**
A: EAFP（先做再处理异常）是 Python 风格；LBYL（先检查再做）是 C 风格。Python 推荐 EAFP。

**Q: 什么时候用 raise 而不是返回错误码？**
A: Python 不用错误码。异常是控制流的一部分。真正"异常"的情况用 raise，正常的分支逻辑用 if/else。
