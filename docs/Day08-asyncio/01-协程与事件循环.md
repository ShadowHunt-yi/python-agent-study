# 01 — 协程与事件循环

## 一、事件循环是什么？

事件循环（Event Loop）是异步编程的核心——一个无限循环，不断检查有没有待执行的任务。

```
┌─────────────────────────────────────┐
│           Event Loop                │
│  ┌─────────┐  ┌─────────┐  ┌─────┐ │
│  │ Task A  │  │ Task B  │  │ ... │ │
│  │(等待API) │  │(等待DB) │  │     │ │
│  └────┬────┘  └────┬────┘  └─────┘ │
│       │            │                │
│   I/O 完成      I/O 完成            │
│       ↓            ↓                │
│   继续执行      继续执行             │
└─────────────────────────────────────┘
```

### 事件循环调度机制

```
事件循环每一轮（称为 "tick"）的工作：

1. 执行所有已就绪的回调（scheduled callbacks）
2. 执行已完成的 I/O 回调
3. 执行到期的定时器回调（sleep 到期的任务）
4. 如果没有任务了 → 退出循环

关键：协程遇到 await 时，控制权交还给事件循环，
事件循环可以去执行其他就绪的任务，等 I/O 完成后再回来继续。
```

```python
# 调度顺序示例
async def main():
    print("A")                    # 立即执行
    await asyncio.sleep(0)        # 让出控制权，允许其他任务执行
    print("B")                    # 下一轮 tick 执行
    await asyncio.sleep(1)        # 让出 1 秒
    print("C")                    # 1 秒后执行

# 输出：A → (让出) → B → (等 1s) → C
```

**Python vs JS 的事件循环：**
- JS：浏览器/Node.js 自动启动事件循环，你写的 `async` 函数自动被调度
- Python：需要显式启动 `asyncio.run(main())`，不调用就不会运行
- Python 的 `asyncio.run()` **不能在已有循环中调用**（会报 `RuntimeError`）
- JS 的微任务（Promise.then）优先级高于宏任务（setTimeout）；Python 没有这种区分，都是按就绪顺序执行

## 二、协程基础

```python
import asyncio

# async def 定义协程函数
async def greet(name: str) -> str:
    await asyncio.sleep(1)    # 模拟 I/O
    return f"Hello, {name}"

# 运行协程
result = asyncio.run(greet("Alice"))
# asyncio.run() 做了两件事：创建事件循环 + 运行协程直到完成
```

## 二、Task —— 并发执行

```python
import asyncio

async def fetch(url: str, delay: float) -> str:
    await asyncio.sleep(delay)
    return f"{url} done"

async def main():
    # 创建 Task —— 立即开始执行
    task1 = asyncio.create_task(fetch("api/a", 2))
    task2 = asyncio.create_task(fetch("api/b", 1))

    # 两个任务并发执行，总耗时 2s（不是 3s）
    result1 = await task1
    result2 = await task2
    print(result1, result2)

asyncio.run(main())
```

```javascript
// JS 对比
async function main() {
    const [r1, r2] = await Promise.all([
        fetch("api/a"),
        fetch("api/b"),
    ]);
}
```

## 三、gather —— 批量并发

```python
async def main():
    results = await asyncio.gather(
        fetch("api/a", 2),
        fetch("api/b", 1),
        fetch("api/c", 3),
    )
    print(results)  # ['api/a done', 'api/b done', 'api/c done']
    # 总耗时 3s（最慢的那个）
```

### gather 的异常处理

```python
async def main():
    # return_exceptions=True —— 不抛异常，把异常作为结果返回
    results = await asyncio.gather(
        fetch("api/a", 1),
        fetch("api/b", 1),  # 假设这个会抛异常
        return_exceptions=True,
    )
    for r in results:
        if isinstance(r, Exception):
            print(f"错误: {r}")
        else:
            print(f"成功: {r}")
```

## 三-B、TaskGroup —— 结构化并发（Python 3.11+）

```python
async def main():
    # TaskGroup 比 gather 更安全：一个任务失败自动取消其他任务
    async with asyncio.TaskGroup() as tg:
        task1 = tg.create_task(fetch("a", 2))
        task2 = tg.create_task(fetch("b", 1))
        task3 = tg.create_task(fetch("c", 3))

    # 所有任务完成后才能到这里
    print(task1.result(), task2.result(), task3.result())

# 如果 task2 抛异常：
# - gather 会等待其他任务完成（可能浪费时间）
# - TaskGroup 会立即取消 task1 和 task3，抛出 ExceptionGroup
```

### TaskGroup vs gather

```python
# gather —— 所有任务都执行，收集结果
results = await asyncio.gather(*tasks, return_exceptions=True)

# TaskGroup —— 结构化并发，快速失败
async with asyncio.TaskGroup() as tg:
    for url in urls:
        tg.create_task(fetch(url))
# 任何一个任务失败 → 其他任务自动取消 → 抛出 ExceptionGroup
```

### 处理 ExceptionGroup

```python
async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(might_fail_1())
            tg.create_task(might_fail_2())
    except* ValueError as eg:
        # except* 是 Python 3.11+ 语法，处理 ExceptionGroup
        for exc in eg.exceptions:
            print(f"ValueError: {exc}")
    except* TypeError as eg:
        for exc in eg.exceptions:
            print(f"TypeError: {exc}")
```

### except* 详解

```python
# except* 是 Python 3.11+ 新语法，专门处理 ExceptionGroup
# 普通 except 不能捕获 ExceptionGroup 中的单个异常

# ExceptionGroup = 多个异常的容器（TaskGroup 失败时产生）
# 一个 ExceptionGroup 可能同时包含 ValueError 和 TypeError

# except* 会"拆分" ExceptionGroup，按类型分别处理
try:
    raise ExceptionGroup("errors", [
        ValueError("bad value"),
        TypeError("bad type"),
        ValueError("another bad value"),
    ])
except* ValueError as eg:
    # eg 只包含 ValueError 类型的异常
    print(f"捕获到 {len(eg.exceptions)} 个 ValueError")
except* TypeError as eg:
    # eg 只包含 TypeError 类型的异常
    print(f"捕获到 {len(eg.exceptions)} 个 TypeError")

# 输出：
# 捕获到 2 个 ValueError
# 捕获到 1 个 TypeError

# 关键区别：
# except ValueError:    → 只能捕获单个 ValueError，不能捕获 ExceptionGroup
# except* ValueError:   → 捕获 ExceptionGroup 中所有 ValueError，返回新的 ExceptionGroup
```

### TaskGroup vs gather 选择指南

```python
# 场景 1：所有请求都要尝试，部分失败可以接受
# → 用 gather + return_exceptions=True
results = await asyncio.gather(*tasks, return_exceptions=True)
successes = [r for r in results if not isinstance(r, Exception)]
failures = [r for r in results if isinstance(r, Exception)]

# 场景 2：一个失败就全部取消（银行转账、批量操作）
# → 用 TaskGroup
async with asyncio.TaskGroup() as tg:
    tg.create_task(transfer(from_acc, to_acc_1, amount1))
    tg.create_task(transfer(from_acc, to_acc_2, amount2))
# 任一失败 → 自动回滚（在 except* 中处理）
```

## 四、wait —— 更灵活的等待

```python
async def main():
    tasks = {
        asyncio.create_task(fetch("a", 1)),
        asyncio.create_task(fetch("b", 3)),
        asyncio.create_task(fetch("c", 2)),
    }

    # 等待第一个完成
    done, pending = await asyncio.wait(tasks, return_when=asyncio.FIRST_COMPLETED)
    print(f"第一个完成: {[t.result() for t in done]}")

    # 等待全部完成
    done, pending = await asyncio.wait(tasks, return_when=asyncio.ALL_COMPLETED)

    # 超时
    done, pending = await asyncio.wait(tasks, timeout=2.0)
    print(f"2s 内完成: {len(done)}, 未完成: {len(pending)}")
```

### wait vs gather

```python
# gather —— 简单批量并发，返回结果列表
results = await asyncio.gather(*coroutines)

# wait —— 更灵活，支持 FIRST_COMPLETED / FIRST_EXCEPTION / 超时
done, pending = await asyncio.wait(tasks, return_when=asyncio.FIRST_COMPLETED)
```

## 五、as_completed —— 按完成顺序处理

```python
async def main():
    tasks = [
        asyncio.create_task(fetch("a", 3)),
        asyncio.create_task(fetch("b", 1)),
        asyncio.create_task(fetch("c", 2)),
    ]

    # 按完成顺序处理（不按提交顺序）
    for coro in asyncio.as_completed(tasks):
        result = await coro
        print(f"完成: {result}")
    # 输出顺序: b(1s) → c(2s) → a(3s)
```

## 六、超时处理

```python
import asyncio

async def slow_operation():
    await asyncio.sleep(100)
    return "done"

async def main():
    try:
        # 方式 1：asyncio.timeout（3.11+）
        async with asyncio.timeout(5):
            result = await slow_operation()
    except TimeoutError:
        print("超时了")

    # 方式 2：wait_for
    try:
        result = await asyncio.wait_for(slow_operation(), timeout=5)
    except TimeoutError:
        print("超时了")
```

## 七、Semaphore —— 控制并发数

```python
async def fetch_with_limit(sem: asyncio.Semaphore, url: str):
    async with sem:  # 超过限制时等待
        return await fetch(url, 1)

async def main():
    sem = asyncio.Semaphore(10)  # 最多 10 个并发
    urls = [f"http://api.example.com/{i}" for i in range(100)]

    tasks = [fetch_with_limit(sem, url) for url in urls]
    results = await asyncio.gather(*tasks)
```

## 八、Lock —— 互斥锁

```python
async def main():
    lock = asyncio.Lock()
    counter = 0

    async def increment():
        nonlocal counter  # 声明使用外层函数的变量（JS 不需要这步，闭包自动捕获）
        async with lock:
            temp = counter
            await asyncio.sleep(0)  # 让出控制权
            counter = temp + 1

    await asyncio.gather(*[increment() for _ in range(100)])
    print(counter)  # 100（不会出现竞态条件）
```

## 九、Queue —— 异步队列

```python
async def producer_consumer():
    queue = asyncio.Queue(maxsize=10)

    async def producer():
        for i in range(20):
            await queue.put(i)
            print(f"生产: {i}")
        # 毒丸（Poison Pill）模式：放入 None 通知消费者退出
        # 有多个消费者时，每个消费者收到毒丸后要重新放回，让其他消费者也能收到
        await queue.put(None)

    async def consumer(name: str):
        while True:
            item = await queue.get()
            if item is None:
                await queue.put(None)  # 重新放回，让其他消费者也能收到
                break
            await asyncio.sleep(0.1)
            print(f"{name} 消费: {item}")
            queue.task_done()

    await asyncio.gather(
        producer(),
        consumer("C1"),
        consumer("C2"),
    )
```

## 十二、面试高频问题

**Q: asyncio.run() 做了什么？**
A: 创建新的事件循环，运行传入的协程，完成后关闭循环。

**Q: Task 和协程的区别？**
A: 协程（coroutine）是惰性的，不主动执行。Task 是对协程的包装，创建后立即调度执行。

**Q: gather 和 wait 的区别？**
A: gather 返回结果列表（按传入顺序）。wait 返回 (done, pending) 集合，支持 FIRST_COMPLETED 和超时。

**Q: 为什么需要 Semaphore？**
A: 防止并发数过高导致服务器过载或连接池耗尽。

**Q: async/await 和 JS 的有什么区别？**
A: 语法几乎相同。但 Python 的 asyncio 是单线程协作式，JS 的 Promise 也是单线程。Python 有 GIL，但 I/O 密集型任务不受影响。

**Q: TaskGroup 和 gather 怎么选？**
A: 需要快速失败（一个失败全部取消）用 TaskGroup。需要所有结果（部分失败可接受）用 gather + return_exceptions=True。

**Q: except* 和 except 的区别？**
A: except 捕获单个异常。except* 是 Python 3.11+ 语法，专门拆分 ExceptionGroup，按类型分别处理。TaskGroup 失败时产生 ExceptionGroup，必须用 except* 处理。

**Q: 事件循环的调度顺序？**
A: 每轮 tick：就绪回调 → I/O 回调 → 定时器回调。协程遇到 await 让出控制权，事件循环去执行其他就绪任务。没有 JS 那样的微任务/宏任务优先级区分。

**Q: await asyncio.sleep(0) 有什么用？**
A: 不实际等待，但让出控制权给事件循环，允许其他就绪的任务执行。常用于协作式让出。
