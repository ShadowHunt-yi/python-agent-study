# 02 — 验证器

## 一、field_validator

```python
from pydantic import BaseModel, field_validator, ValidationInfo

class User(BaseModel):
    name: str
    age: int
    email: str

    @field_validator("name")
    @classmethod
    def name_must_be_capitalized(cls, v: str, info: ValidationInfo) -> str:
        if not v[0].isupper():
            raise ValueError("名称必须大写开头")
        return v.strip()

    @field_validator("age")
    @classmethod
    def age_must_be_positive(cls, v: int) -> int:
        if v < 0:
            raise ValueError("年龄不能为负")
        return v

# 验证多个字段
@field_validator("name", "email")
@classmethod
def not_empty(cls, v: str) -> str:
    if not v.strip():
        raise ValueError("不能为空")
    return v.strip()
```

### mode 参数

```python
# mode="before" —— 在类型转换之前验证
@field_validator("name")
@classmethod
def validate_name(cls, v):
    # v 此时还是原始输入（可能是任何类型）
    if not isinstance(v, str):
        raise ValueError("必须是字符串")
    return v

# mode="after"（默认）—— 在类型转换之后验证
@field_validator("age")
@classmethod
def validate_age(cls, v: int) -> int:
    # v 已经是 int
    if v < 0:
        raise ValueError("不能为负")
    return v
```

## 二、model_validator

```python
from pydantic import BaseModel, model_validator

class Order(BaseModel):
    items: list[str]
    quantities: list[int]
    discount: float = 0.0

    @model_validator(mode="after")
    def check_lists_same_length(self) -> "Order":
        if len(self.items) != len(self.quantities):
            raise ValueError("items 和 quantities 长度必须相同")
        return self

    @model_validator(mode="after")
    def check_discount(self) -> "Order":
        if self.discount > 1.0:
            raise ValueError("折扣不能超过 100%")
        return self
```

### mode="before" —— 预处理原始数据

```python
class Config(BaseModel):
    data: dict

    @model_validator(mode="before")
    @classmethod
    def parse_raw(cls, values):
        # values 是原始输入（可能是 JSON 字符串）
        if isinstance(values, str):
            import json
            values = {"data": json.loads(values)}
        return values

# 支持 dict 和 JSON 字符串输入
config1 = Config(data={"key": "value"})
config2 = Config.model_validate('{"key": "value"}')
```

## 三、Annotated 类型约束

```python
from typing import Annotated
from pydantic import BaseModel, Field, StringConstraints

class User(BaseModel):
    name: Annotated[str, StringConstraints(min_length=1, max_length=50)]
    age: Annotated[int, Field(ge=0, le=150)]
    email: Annotated[str, Field(pattern=r"^[\w.-]+@[\w.-]+\.\w+$")]
    bio: Annotated[str, Field(max_length=500)] = ""
```

## 四、computed_field

```python
from pydantic import BaseModel, computed_field

class Rectangle(BaseModel):
    width: float
    height: float

    @computed_field
    @property
    def area(self) -> float:
        return self.width * self.height

    @computed_field
    @property
    def perimeter(self) -> float:
        return 2 * (self.width + self.height)

r = Rectangle(width=5, height=3)
print(r.model_dump())
# {'width': 5.0, 'height': 3.0, 'area': 15.0, 'perimeter': 16.0}
```

## 五、TypeAdapter —— 非模型验证

```python
from pydantic import TypeAdapter

# 验证基本类型
ta_int = TypeAdapter(int)
ta_int.validate_python("42")        # 42
ta_int.validate_python(42)          # 42

# 验证列表
ta_list = TypeAdapter(list[int])
ta_list.validate_python(["1", "2", "3"])  # [1, 2, 3]

# 验证字典
ta_dict = TypeAdapter(dict[str, int])
ta_dict.validate_python({"a": "1", "b": "2"})  # {'a': 1, 'b': 2}

# JSON Schema
ta_list.json_schema()  # {"items": {"type": "integer"}, "type": "array"}
```

## 六、严格模式 vs 宽松模式

```python
from pydantic import BaseModel, ValidationError

class User(BaseModel):
    id: int
    name: str

# 宽松模式（默认）—— 自动类型转换
user = User.model_validate({"id": "123", "name": "Alice"})  # id → 123

# 严格模式 —— 不转换
try:
    User.model_validate({"id": "123", "name": "Alice"}, strict=True)
except ValidationError as e:
    print(e)  # "Input should be a valid integer"
```

## 七、序列化选项

```python
from pydantic import BaseModel, ConfigDict

class User(BaseModel):
    model_config = ConfigDict(serialize_by_alias=True)

    name: str
    internal_id: str = Field(alias="id", serialization_alias="user_id")

user = User(name="Alice", id="abc123")
user.model_dump()            # {'name': 'Alice', 'user_id': 'abc123'}
user.model_dump(by_alias=True)  # 同上

# 排除选项
user.model_dump(exclude={"internal_id"})
user.model_dump(include={"name"})
user.model_dump(exclude_unset=True)    # 只输出设置过的字段
user.model_dump(exclude_defaults=True) # 排除等于默认值的字段
```

## 八、面试高频问题

**Q: `@field_validator` 的 mode="before" 和 mode="after" 有什么区别？**
A: "before" 在类型转换之前执行（拿到原始输入），"after" 在转换之后执行（拿到目标类型）。

**Q: `@model_validator` 和 `@field_validator` 的区别？**
A: `field_validator` 验证单个字段，`model_validator` 验证整个模型（可以访问所有字段，做跨字段校验）。

**Q: TypeAdapter 什么时候用？**
A: 验证非 BaseModel 的类型（如 `list[int]`、`dict[str, Any]`）。不需要定义类就能校验数据。

**Q: computed_field 的作用？**
A: 定义派生字段，自动包含在序列化结果中。类似 `@property` 但参与 JSON 序列化。
