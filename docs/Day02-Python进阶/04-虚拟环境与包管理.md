# 04 — 虚拟环境与包管理

## 一、venv —— 内置虚拟环境

```bash
# 创建虚拟环境
python -m venv venv

# 激活（Windows）
venv\Scripts\activate

# 激活（Linux/macOS）
source venv/bin/activate

# 确认激活（命令行前出现 (venv) 标识）
which python    # 指向 venv 中的 python

# 退出
deactivate

# 删除虚拟环境（直接删目录）
rm -rf venv/
```

### 为什么需要虚拟环境

```
# 项目 A 需要 requests==2.28
# 项目 B 需要 requests==2.31
# 没有虚拟环境，全局只能装一个版本

# 虚拟环境 = 每个项目独立的 Python 环境
# 类似 JS 的 node_modules，但更彻底
```

## 二、pip —— 包管理器

```bash
# 安装
pip install requests
pip install requests==2.31.0       # 指定版本
pip install "requests>=2.28,<3.0"  # 版本范围
pip install requests flask         # 多个包

# 从文件安装
pip install -r requirements.txt

# 升级
pip install --upgrade requests

# 卸载
pip uninstall requests

# 查看已安装
pip list
pip show requests      # 详细信息

# 导出依赖
pip freeze > requirements.txt

# 检查过期
pip list --outdated
```

### 版本约束语法

```
flask==3.0.0       # 精确版本
flask>=3.0.0       # 最低版本
flask<4.0.0        # 最高版本
flask>=3.0,<4.0    # 范围
flask~=3.0         # 兼容版本（>=3.0, <4.0）
flask!=3.0.1       # 排除版本
```

## 四、pyproject.toml（现代标准）

```toml
[project]
name = "my-agent"
version = "0.1.0"
description = "An AI agent built with Python"
requires-python = ">=3.10"
dependencies = [
    "openai>=1.0.0",
    "langchain>=0.3.0",
    "langgraph>=1.0.0",
    "pydantic>=2.0.0",
    "fastapi>=0.115.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0",
    "pytest-asyncio>=0.23.0",
    "ruff>=0.1.0",
    "mypy>=1.0",
]

[build-system]
requires = ["setuptools>=68.0"]
build-backend = "setuptools.backends._legacy:_Backend"
```

```bash
# 使用 pyproject.toml
pip install .                # 安装项目
pip install -e .             # 可编辑安装（开发模式）
pip install -e ".[dev]"      # 包含开发依赖
```

## 五、包管理工具对比

| 工具 | 命令 | 锁文件 | 速度 |
|------|------|--------|------|
| pip | `pip install` | requirements.txt（手动） | 慢 |
| pip-tools | `pip-compile` | requirements.txt（自动生成） | 中 |
| Poetry | `poetry add` | poetry.lock | 中 |
| **uv** | `uv pip install` | uv.lock | **极快**（Rust） |

### uv —— 推荐

```bash
# 安装 uv
pip install uv

# 创建虚拟环境 + 安装
uv venv
uv pip install fastapi openai pydantic

# 从 pyproject.toml 安装
uv pip install -e ".[dev]"

# uv 速度是 pip 的 10-100 倍
```

## 七、面试高频问题

**Q: 为什么需要虚拟环境？**
A: 隔离项目依赖，避免版本冲突。类似 JS 的 node_modules，但 Python 的全局安装会影响所有项目。

**Q: pyproject.toml 和 requirements.txt 的区别？**
A: `pyproject.toml` 是项目元数据 + 依赖声明的标准格式。`requirements.txt` 是 pip 的锁文件格式。开发用 `pyproject.toml`，部署用 `requirements.txt`。

**Q: 推荐什么包管理工具？**
A: 新项目用 `uv`（极快、Rust 实现）。已有项目用 `pip` 或 `Poetry`。
