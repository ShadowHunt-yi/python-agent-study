# 01 — 变量与数据类型

## 一、变量声明

Python 不需要 `let`/`const`/`var`，直接赋值即可。

```python
# Python
name = "Alice"        # 无需关键字
age = 30
is_active = True
PI = 3.14159          # 纯大写是常量的约定（不强制）
```

```javascript
// JavaScript 对比
let name = "Alice";
const age = 30;
let isActive = true;
const PI = 3.14159;
```

### JS 开发者注意

| 差异 | Python | JS |
|------|--------|-----|
| 常量 | 无 `const`，纯大写命名仅是约定 | `const` 关键字 |
| 命名风格 | `snake_case`（约定） | `camelCase`（约定） |
| 作用域 | 无块级作用域，`if`/`for` 内变量泄漏到函数 | `let`/`const` 有块级作用域 |
| 语句末尾 | 无需分号 | 需要分号（或 ASI） |

```python
# 块级作用域陷阱
if True:
    x = 10
print(x)  # 10 —— x 在 if 外依然可用
```

## 二、数据类型对比

| Python | JavaScript | 说明 |
|--------|-----------|------|
| `int` | `Number` | Python 整数无精度上限（无 `MAX_SAFE_INTEGER`） |
| `float` | `Number` | Python 区分 int 和 float |
| `str` | `String` | 都是不可变 |
| `bool`（`True`/`False`） | `boolean`（`true`/`false`） | **首字母大写不同** |
| `None` | `null` / `undefined` | Python 只有 `None`，没有 `undefined` |
| `list` | `Array` | 都可变 |
| `dict` | `Object` / `Map` | Python 3.7+ 保持插入顺序 |
| `tuple` | — | 不可变序列，JS 无对应 |
| `set` | `Set` | 概念相似 |
| `bytes` | `Uint8Array` | 原始字节 |

```python
# 类型检查
type(42)              # <class 'int'>
isinstance(42, int)   # True  —— 推荐用 isinstance
isinstance("hi", str) # True
```

```javascript
// JavaScript
typeof 42             // "number"
42 instanceof Number  // false（原始类型 vs 对象）
```

### 关键陷阱

```python
# 1. 布尔值首字母大写
active = True    # ✓
active = true    # ✗ NameError: name 'true' is not defined

# 2. None 比较用 is，不用 ==
x = None
x == None   # True（能用但不推荐）
x is None   # True（推荐 —— 更快、更明确）

# 3. Python 没有 undefined
# 未定义的变量直接 NameError，不是 undefined
```

## 三、运算符

### 算术运算符

```python
2 ** 3      # 8    —— 幂运算（JS: 2 ** 3 或 Math.pow(2, 3)）
17 // 5     # 3    —— 整除（JS: Math.floor(17 / 5)）
17 % 5      # 2    —— 取余（同 JS）
-17 // 5    # -4   —— 向下取整（JS: Math.floor(-17/5) 也是 -4）
```

### 逻辑运算符

```python
# Python 用英文单词，不是符号
True and False    # JS: true && false
True or False     # JS: true || false
not True          # JS: !true

# 短路求值同 JS
x = None
value = x or "default"   # "default"（x 为 falsy 时取后者）
```

### 身份 vs 相等

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

a == b      # True  —— 值相等（类似 JS ===）
a is b      # False —— 不是同一个对象（JS 无直接对应）
a is c      # True  —— 同一个对象
a is not b  # True
```

| 运算符 | Python | JS |
|--------|--------|-----|
| 值相等 | `==` | `===` |
| 引用相同 | `is` | 无（需手动比较） |
| 不等 | `!=` | `!==` |
| 逻辑与 | `and` | `&&` |
| 逻辑或 | `or` | `\|\|` |
| 逻辑非 | `not` | `!` |

### 成员运算符

```python
# in / not in
fruits = ["apple", "banana", "cherry"]
"apple" in fruits       # True
"grape" not in fruits   # True

user = {"name": "Alice", "age": 30}
"name" in user          # True（检查 key）
"Alice" in user         # False
```

```javascript
// JS
fruits.includes("apple");
"name" in user;          // 检查属性
user.hasOwnProperty("name");
```

### 海象运算符 := （Python 3.8+）

```python
# 赋值并返回值，常用于 while 和 if
while (line := input("Enter: ")) != "quit":
    print(f"You said: {line}")

# 在推导式中避免重复计算
results = [y for x in data if (y := process(x)) > threshold]
```

## 四、类型转换

```python
int("42")        # 42
int(3.9)         # 3（截断，不是四舍五入）
float("3.14")    # 3.14
str(42)          # "42"
bool(0)          # False
bool("")         # False
bool([])         # False
bool(None)       # False
bool(1)          # True
bool("hi")       # True

# 容器转换
list("hello")           # ['h', 'e', 'l', 'l', 'o']
tuple([1, 2, 3])        # (1, 2, 3)
set([1, 1, 2, 2])       # {1, 2}
dict([("a", 1), ("b", 2)])  # {'a': 1, 'b': 2}
list(range(5))           # [0, 1, 2, 3, 4]
```


## 五、面试高频问题

**Q: Python 和 JS 最大的语法区别是什么？**
A: 缩进代替花括号、无块级作用域、`True`/`False`/`None` 首字母大写、用 `and`/`or`/`not` 代替 `&&`/`||`/`!`。

**Q: `==` 和 `is` 的区别？**
A: `==` 比较值是否相等，`is` 比较是否是同一个对象（内存地址）。判断 `None` 时用 `is None`。

**Q: Python 有 undefined 吗？**
A: 没有。只有 `None`。未定义的变量直接抛 `NameError`。

**Q: Python 的整数有精度限制吗？**
A: 没有。Python 的 `int` 是任意精度的，可以表示任意大的整数。
