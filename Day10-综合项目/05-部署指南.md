# 05 — 部署指南

## 一、Docker 部署

```dockerfile
# Dockerfile
FROM python:3.12-slim

WORKDIR /app

# 安装依赖
COPY pyproject.toml .
RUN pip install --no-cache-dir .

# 复制代码
COPY src/ src/

# 暴露端口
EXPOSE 8000

# 启动
CMD ["uvicorn", "mini_agent.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

```bash
# 构建和运行
docker build -t mini-agent .
docker run -p 8000:8000 --env-file .env mini-agent
```

## 二、docker-compose（带数据库）

```yaml
# docker-compose.yml
version: "3.8"
services:
  agent:
    build: .
    ports:
      - "8000:8000"
    env_file:
      - .env
    depends_on:
      - redis
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

## 四、Gunicorn + Uvicorn（生产）

```bash
# 安装
pip install gunicorn uvicorn

# 启动（多 worker）
gunicorn mini_agent.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000 \
  --timeout 120
```

## 七、监控指标

```python
from fastapi import Request
import time

@app.middleware("http")
async def metrics(request: Request, call_next):
    start = time.perf_counter()
    response = await call_next(request)
    elapsed = time.perf_counter() - start

    # Prometheus 格式
    REQUEST_LATENCY.labels(
        method=request.method,
        endpoint=request.url.path,
        status=response.status_code,
    ).observe(elapsed)

    return response
```

## 八、CI/CD 流程

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install -e ".[dev]"
      - run: pytest tests/ -v

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build and push Docker image
        run: |
          docker build -t mini-agent .
          docker push mini-agent
```

## 九、面试高频问题

**Q: 怎么部署 FastAPI 应用？**
A: Docker + Gunicorn + Uvicorn Worker。Gunicorn 管理多个 Uvicorn Worker 进程。

**Q: 生产环境需要哪些配置？**
A: 多 Worker、健康检查、日志收集、限流、CORS、环境变量管理、CI/CD。

**Q: 怎么管理 API Key？**
A: 不要硬编码。用环境变量或 Secrets Manager。Docker 用 `--env-file` 或 `docker secrets`。
