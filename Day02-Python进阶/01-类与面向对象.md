# 01 — 类与面向对象

## 一、基础类

```python
class Dog:
    # 类属性（所有实例共享）
    species = "Canis familiaris"

    def __init__(self, name: str, age: int):
        # 实例属性
        self.name = name    # self 是显式的（类似 JS 的 this）
        self.age = age

    def bark(self) -> str:   # self 必须是第一个参数
        return f"{self.name} says Woof!"

    def __str__(self) -> str:     # print() 时调用
        return f"Dog({self.name}, {self.age})"

    def __repr__(self) -> str:    # 交互式/调试时调用
        return f"Dog(name='{self.name}', age={self.age})"

dog = Dog("Rex", 5)
print(dog)           # Dog(Rex, 5) —— 调用 __str__
dog.bark()           # "Rex says Woof!"
```

```javascript
// JS 对比
class Dog {
    static species = "Canis familiaris";
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    bark() { return `${this.name} says Woof!`; }
    toString() { return `Dog(${this.name}, ${this.age})`; }
}
```

### 关键差异

| 差异 | Python | JS |
|------|--------|-----|
| 构造函数 | `__init__(self)` | `constructor()` |
| this/self | `self` 显式第一个参数 | `this` 隐式 |
| 创建属性 | `self.name = name`（必须显式） | `this.name = name` |
| 字符串表示 | `__str__` / `__repr__` | `toString()` |

## 二、继承

```python
class Animal:
    def __init__(self, name: str):
        self.name = name

    def speak(self) -> str:
        raise NotImplementedError

class Cat(Animal):
    def __init__(self, name: str, indoor: bool = True):
        super().__init__(name)    # 调用父类构造函数
        self.indoor = indoor

    def speak(self) -> str:
        return f"{self.name} says Meow!"

# 多重继承
class PetMixin:
    def is_pet(self) -> bool:
        return True

class HouseCat(Cat, PetMixin):
    pass

cat = HouseCat("Whiskers")
cat.speak()      # "Whiskers says Meow!"
cat.is_pet()     # True
```

### MRO（方法解析顺序）

```python
# Python 使用 C3 线性化算法确定方法调用顺序
print(HouseCat.__mro__)
# (<class 'HouseCat'>, <class 'Cat'>, <class 'Animal'>, <class 'PetMixin'>, <class 'object'>)
```

## 三、@property

```python
class Circle:
    def __init__(self, radius: float):
        self._radius = radius   # 下划线前缀 = "私有"（约定）

    @property
    def radius(self) -> float:
        """getter"""
        return self._radius

    @radius.setter
    def radius(self, value: float):
        """setter"""
        if value < 0:
            raise ValueError("半径不能为负")
        self._radius = value

    @property
    def area(self) -> float:
        """计算属性（只读）"""
        import math
        return math.pi * self._radius ** 2

c = Circle(5)
print(c.area)       # 78.54 —— 像属性一样访问
c.radius = 10       # 调用 setter
# c.area = 100      # AttributeError: 不能设置
```

```javascript
// JS
class Circle {
    #radius;
    constructor(radius) { this.#radius = radius; }
    get radius() { return this.#radius; }
    set radius(v) { if (v < 0) throw new Error("..."); this.#radius = v; }
    get area() { return Math.PI * this.#radius ** 2; }
}
```

## 四、@classmethod 和 @staticmethod

```python
class Date:
    def __init__(self, year: int, month: int, day: int):
        self.year = year
        self.month = month
        self.day = day

    @classmethod
    def from_string(cls, date_string: str) -> "Date":
        """工厂方法 —— 接收类本身"""
        year, month, day = map(int, date_string.split("-"))
        return cls(year, month, day)    # cls 是类本身

    @staticmethod
    def is_valid(date_string: str) -> bool:
        """工具函数 —— 不接收 self 或 cls"""
        parts = date_string.split("-")
        return len(parts) == 3

d = Date.from_string("2024-01-15")   # 工厂方法
Date.is_valid("2024-01-15")           # True
```

| 装饰器 | 第一个参数 | 用途 |
|--------|-----------|------|
| 普通方法 | `self`（实例） | 操作实例数据 |
| `@classmethod` | `cls`（类） | 工厂方法、替代构造函数 |
| `@staticmethod` | 无 | 工具函数，与类/实例无关 |

## 五、魔术方法（Dunder Methods）

```python
class Vector:
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

    # 运算符重载
    def __add__(self, other: "Vector") -> "Vector":
        return Vector(self.x + other.x, self.y + other.y)

    def __mul__(self, scalar: float) -> "Vector":
        return Vector(self.x * scalar, self.y * scalar)

    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Vector):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    # 容器协议
    def __len__(self) -> int:
        return 2

    def __getitem__(self, index: int) -> float:
        if index == 0: return self.x
        if index == 1: return self.y
        raise IndexError

    # 字符串表示
    def __str__(self) -> str:
        return f"({self.x}, {self.y})"

    def __repr__(self) -> str:
        return f"Vector({self.x}, {self.y})"

    # 可哈希（需要 __hash__）
    def __hash__(self) -> int:
        return hash((self.x, self.y))

v1 = Vector(1, 2)
v2 = Vector(3, 4)
v3 = v1 + v2       # Vector(4, 6)
v1 * 3              # Vector(3, 6)
v1 == Vector(1, 2)  # True
len(v1)             # 2
v1[0]               # 1
```

### 常用魔术方法速查

| 方法 | 触发方式 | 说明 |
|------|---------|------|
| `__init__` | `Class()` | 构造 |
| `__str__` | `str(obj)` / `print(obj)` | 用户友好的字符串 |
| `__repr__` | `repr(obj)` / 交互式 | 开发者友好的字符串 |
| `__len__` | `len(obj)` | 长度 |
| `__getitem__` | `obj[key]` | 索引访问 |
| `__setitem__` | `obj[key] = val` | 索引设置 |
| `__contains__` | `x in obj` | 成员检测 |
| `__iter__` | `for x in obj` | 迭代 |
| `__call__` | `obj()` | 可调用 |
| `__add__` | `a + b` | 加法 |
| `__eq__` | `a == b` | 相等比较 |
| `__hash__` | `hash(obj)` | 哈希值 |
| `__enter__`/`__exit__` | `with obj` | 上下文管理器 |
| `__aenter__`/`__aexit__` | `async with obj` | 异步上下文管理器 |

## 六、dataclass（Python 3.7+）

```python
from dataclasses import dataclass, field

@dataclass
class User:
    name: str
    age: int
    email: str = ""                          # 默认值
    tags: list = field(default_factory=list) # 可变默认值

    def is_adult(self) -> bool:
        return self.age >= 18

# 自动生成 __init__, __repr__, __eq__
user = User("Alice", 30)
print(user)    # User(name='Alice', age=30, email='', tags=[])
```

### dataclass 选项

```python
# 不可变（frozen）
@dataclass(frozen=True)
class Point:
    x: float
    y: float

p = Point(3.0, 4.0)
# p.x = 5    # FrozenInstanceError

# 可排序
@dataclass(order=True)
class Student:
    grade: float
    name: str

students = [Student(3.9, "Alice"), Student(3.7, "Bob")]
sorted(students)  # 先按 grade，再按 name

# 不生成 __init__（自定义）
@dataclass(init=False)
class Config:
    data: dict

    def __init__(self, **kwargs):
        self.data = kwargs

# 继承
@dataclass
class Animal:
    name: str

@dataclass
class Dog(Animal):
    breed: str = "Unknown"

dog = Dog("Rex", "Labrador")  # 或 Dog(name="Rex", breed="Labrador")
```

### dataclass vs Pydantic BaseModel

| 特性 | dataclass | Pydantic |
|------|-----------|----------|
| 运行时校验 | 无 | 有 |
| 序列化 | 无内置 | `model_dump()` / `model_dump_json()` |
| JSON Schema | 无 | `model_json_schema()` |
| 性能 | 快 | Rust 核心更快 |
| 使用场景 | 简单数据容器 | API 输入/输出、配置 |

## 八、typing.Protocol —— 结构化子类型

```python
from typing import Protocol, runtime_checkable

# 定义接口（类似 TypeScript interface）
@runtime_checkable
class Tool(Protocol):
    name: str
    def invoke(self, args: dict) -> str: ...

# 任何有 name 属性和 invoke 方法的类都自动满足 Tool 协议
class Calculator:
    name = "calculator"
    def invoke(self, args: dict) -> str:
        return str(eval(args["expression"]))

class Searcher:
    name = "search"
    def invoke(self, args: dict) -> str:
        return f"搜索结果: {args['query']}"

# 不需要继承！只要结构匹配即可
def execute_tool(tool: Tool, args: dict) -> str:
    return tool.invoke(args)   # 类型检查通过

execute_tool(Calculator(), {"expression": "2+3"})   # OK
execute_tool(Searcher(), {"query": "Python"})       # OK

# 运行时检查
isinstance(Calculator(), Tool)   # True
```

### Protocol vs ABC

| 特性 | Protocol | ABC |
|------|----------|-----|
| 检查方式 | 结构化（鸭子类型） | 名义性（必须继承） |
| 继承要求 | 不需要 | 必须继承 |
| 灵活性 | 高（适配已有代码） | 低（需要修改类） |
| 使用场景 | 定义工具接口、依赖注入 | 强制继承关系 |

## 九、面试高频问题

**Q: `__str__` 和 `__repr__` 的区别？**
A: `__str__` 面向用户（print 时调用），`__repr__` 面向开发者（调试时调用）。如果没有 `__str__`，回退到 `__repr__`。

**Q: @classmethod 和 @staticmethod 什么时候用？**
A: `@classmethod` 用于工厂方法（如 `Date.from_string()`），因为它能访问类本身。`@staticmethod` 用于与类相关但不需要访问类/实例的工具函数。

**Q: dataclass 的 `field(default_factory=list)` 为什么需要？**
A: 因为 `tags: list = []` 会在所有实例间共享同一个列表（可变默认参数陷阱）。`default_factory` 每次创建新实例时调用 `list()` 生成新列表。

**Q: Python 支持多重继承吗？**
A: 支持，通过 MRO（C3 线性化）解决菱形继承问题。
