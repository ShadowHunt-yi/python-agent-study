## 一、为什么需要异步

```python
# 同步 —— 阻塞
import time

def fetch_sync(url: str) -> str:
    time.sleep(1)  # 模拟网络请求
    return f"data from {url}"

# 3 个请求，耗时 3 秒
start = time.time()
fetch_sync("a")
fetch_sync("b")
fetch_sync("c")
print(f"耗时: {time.time() - start:.1f}s")  # 3.0s
```

```python
# 异步 —— 非阻塞
import asyncio

async def fetch_async(url: str) -> str:
    await asyncio.sleep(1)  # 模拟异步网络请求
    return f"data from {url}"

async def main():
    # 3 个请求，并发执行，耗时约 1 秒
    results = await asyncio.gather(
        fetch_async("a"),
        fetch_async("b"),
        fetch_async("c"),
    )
    return results

start = time.time()
asyncio.run(main())
print(f"耗时: {time.time() - start:.1f}s")  # ~1.0s
```

## 二、async/await 基础

```python
import asyncio

# async def 定义协程函数
async def hello():
    print("Hello")
    await asyncio.sleep(1)  # 让出控制权
    print("World")

# 运行协程
asyncio.run(hello())
```

### await 的含义

```python
async def fetch():
    # await = 暂停当前协程，等待异步操作完成
    # 在等待期间，事件循环可以执行其他协程
    data = await some_async_operation()
    return data
```

## 三、Task —— 并发执行

```python
async def main():
    # create_task —— 立即调度执行
    task1 = asyncio.create_task(fetch_async("a"))
    task2 = asyncio.create_task(fetch_async("b"))

    # 两个任务并发执行
    result1 = await task1
    result2 = await task2
```

## 四、常见异步库

```python
# HTTP
import httpx
async with httpx.AsyncClient() as client:
    resp = await client.get("https://api.example.com")

# 数据库
import aiosqlite
async with aiosqlite.connect("db.sqlite") as db:
    await db.execute("SELECT * FROM users")

# 文件
import aiofiles
async with aiofiles.open("file.txt", "r") as f:
    content = await f.read()
```

```python
class AsyncResource:
    async def __aenter__(self):
        self.conn = await create_connection()

    async def __aexit__(self, *exc):

async with AsyncResource() as conn:

```

## 六、面试高频问题

**Q: Python 的 async/await 和 JavaScript 的有什么区别？**
A: 语法几乎相同。但 Python 的 asyncio 基于事件循环 + 协程（单线程），JS 的 Promise 也是单线程。Python 有 GIL，但 I/O 密集型任务不受影响。

**Q: 什么时候用异步？**
A: I/O 密集型任务（网络请求、数据库查询、文件读写）。CPU 密集型任务用 multiprocessing。

**Q: `asyncio.run()` 做了什么？**
A: 创建新的事件循环，运行传入的协程，完成后关闭循环。通常在程序入口调用一次。
