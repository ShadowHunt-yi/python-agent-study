# 02 — 模块与导入

## 一、基础导入

```python
# 导入整个模块
import math
math.sqrt(16)       # 4.0

# 导入特定名称
from math import sqrt, pi
sqrt(16)            # 4.0（无需 math. 前缀）

# 别名
import numpy as np
from collections import defaultdict as dd

# 导入全部（不推荐 —— 污染命名空间）
from math import *
```

```javascript
// JS 对比
import * as math from "math";
import { sqrt, pi } from "math";
import np from "numpy";
```

### Python 无 `export` —— 模块内所有顶层定义都可被导入

```python
# my_module.py
def public_func():
    pass

def _private_func():    # 下划线前缀 = "私有"（约定）
    pass

# 别人可以导入任何名称
from my_module import public_func
from my_module import _private_func    # 也能导入，但不推荐
```

## 二、包结构

```
my_project/
    main.py
    utils/
        __init__.py      # 使 utils 成为包（可以为空）
        helpers.py
        validators.py
    models/
        __init__.py
        user.py
```

### `__init__.py`

```python
# utils/__init__.py
# 可以为空，也可以控制导出

# 方式一：空文件（最常见）
# 方式二：显式导出
from .helpers import format_date, parse_date
from .validators import validate_email

# 方式三：__all__ 控制 from package import * 的行为
__all__ = ["helpers", "validators"]
```

### 相对导入

```python
# utils/helpers.py 中
from .validators import validate_email        # 同级模块
from ..models.user import User                # 上一级

# main.py 中（顶层脚本不能用相对导入）
from utils.helpers import format_date          # 绝对导入
```

### 导入陷阱

```python
# ✗ 顶层脚本不能用相对导入
# main.py
from .utils.helpers import format_date   # ImportError

# ✓ 应该用绝对导入
from utils.helpers import format_date

# ✓ 或者用 -m 参数运行
# python -m my_project.main
```

## 三、`if __name__ == "__main__"`

```python
# calculator.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

if __name__ == "__main__":
    # 只在直接运行时执行，被导入时不执行
    print(add(3, 4))
```

```javascript
// JS —— 不需要这个模式
// ES 模块的顶层代码只执行一次，但没有这种守卫
```

### 为什么需要

```python
# 如果没有这个守卫：
# calculator.py
def add(a, b):
    return a + b

print(add(3, 4))    # 每次 import 都会执行！

# main.py
import calculator    # 输出 7（不期望的行为）
```

## 七、面试高频问题

**Q: `__init__.py` 的作用是什么？**
A: 标识目录为 Python 包。可以为空，也可以包含初始化代码或控制导出。Python 3.3+ 的命名空间包可以没有它，但最佳实践仍然推荐使用。

**Q: 相对导入和绝对导入的区别？**
A: 相对导入用 `.`（如 `from .module import func`），只能在包内使用。绝对导入用完整路径（如 `from mypackage.module import func`）。顶层脚本只能用绝对导入。

**Q: `if __name__ == "__main__"` 的作用？**
A: 防止模块被 import 时执行测试/演示代码。`__name__` 在直接运行时是 `"__main__"`，被导入时是模块名。
