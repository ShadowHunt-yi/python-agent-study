# Agent 架构速查

## 一、ReAct 循环

```
用户输入
  ↓
LLM 思考 → 是否需要工具？
  ↓ 是              ↓ 否
调用工具          输出回复
  ↓
获取结果
  ↓
反馈给 LLM
  ↓
回到 LLM 思考
```

## 二、核心组件

```
┌─────────────────────────────────┐
│            Agent                │

│  │   LLM    │  │  Tools   │    │
```

## 三、技术栈映射

| 层 | 技术 | 作用 |
|---|------|------|
| API 服务 | FastAPI | 暴露 HTTP 接口 |
| Agent 编排 | LangGraph | 图状态机，条件路由 |
| 工具抽象 | LangChain @tool | 定义工具 schema |
| LLM 调用 | OpenAI SDK | 调用 GPT-4o |
| 数据校验 | Pydantic | 请求/响应/工具参数 |
| 异步并发 | asyncio | 并发工具调用 |
| 测试 | pytest | 单元/集成测试 |

## 四、常见 Agent 模式

### 1. ReAct（最基础）
```
思考 → 行动 → 观察 → 思考 → ...
```

### 2. Plan-and-Execute（规划型）
```
规划 → 执行步骤1 → 执行步骤2 → ... → 汇总
```

### 3. Reflection（反思型）
```
生成 → 自我评估 → 改进 → 评估 → ...
```

### 4. Multi-Agent（多智能体）
```
主管 → 分配任务 → Agent A / Agent B → 汇总
```

## 五、Tool Calling 流程

```
1. 定义工具 schema（JSON Schema）
2. LLM 调用：messages + tools
3. LLM 返回 tool_calls
4. 执行工具函数
5. 结果作为 tool message 发回
6. LLM 整合结果，决定下一步
```

## 六、内存管理

```
短期记忆：对话历史（messages 列表）
长期记忆：向量数据库 / KV 存储
工作记忆：当前任务的中间状态
```

## 七、流式输出

```
LLM 生成 token → SSE 推送 → 前端实时显示
工具调用 → 事件推送 → 前端展示调用过程
最终回复 → 流式输出 → 完成事件
```

## 八、错误处理策略

| 错误 | 处理 |
|------|------|
| LLM 超时 | 指数退避重试 |
| 工具执行失败 | 返回错误信息给 LLM |
| 达到最大步数 | 返回当前结果 |
| Token 超限 | 裁剪对话历史 |
| API 限流 | 退避重试 + 限流器 |
